This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a **Revit BIM Automation Toolkit** — a collection of Dynamo graph files (`.dyn`), Python scripts, and PowerShell graph generators for automating Autodesk Revit workflows. The project targets Korean AEC/BIM professionals.

## Execution Context

There is no traditional build system. Code runs inside Autodesk Revit via Dynamo.

- **Runtime**: Autodesk Revit 2024+ with Dynamo for Revit
- **Python variant**: CPython (via Dynamo Python Script nodes), with `.NET` interop via `clr.AddReference`
- **PowerShell**: 5.1, for generating `.dyn` JSON files from templates
- **No package manager, linter, or test framework** — integration testing is done by running graphs inside Revit

## How to Run Workflows

**Generate a Dynamo graph (PowerShell):**
```powershell
.\scripts\generate_auto_collect_family_dynamo.ps1 -SourcePath "" -OutputPath "path\to\output.dyn" -CollectorScriptPath "path\to\script.py"
.\scripts\generate_shared_parameter_dynamo.ps1 -PythonScriptPath "path\to\script.py" -OutputPath "path\to\output.dyn"
.\scripts\make_dynamo_player_ready.ps1
```

**Run in Revit:**
1. Open Dynamo from Revit → load `.dyn` file → configure input nodes → click Run
2. Or use Dynamo Player for `.dyn` files pre-configured for player execution

## Architecture

### Core Workflow Pattern

```
PowerShell script → generates .dyn JSON → Dynamo loads graph → Python node executes → Revit API operations
```

### Directory Layout

| Path | Purpose |
|------|---------|
| `scripts/` | PowerShell generators that produce `.dyn` files programmatically |
| `#Parameter 기입_충북선/` | Parameter input workflows for a rail project |
| `#SharedParameterValues_Export/` | Excel export of Revit shared parameters |
| `#SharedParameterValues_Import/` | Excel import to Revit shared parameters |
| `#객체 토목에 밀착#/` | Z-axis attachment of families to civil engineering solids |
| `#터널 전등 자동 배치#/` | Automatic tunnel lighting placement |
| `#속성정보 삭제/` | Property information deletion |
| `#ifc_class_standard_by_category/` | IFC class standardization by category |
| `revit_beginner_presentation/` | BIM training slides and speaker notes |

Folders prefixed with `#` are workflow modules; files prefixed with `@` within them are supporting Python utility scripts.

### PowerShell Graph Generators (`scripts/`)

Each script follows the same pattern: accept path arguments → build a dictionary of Dynamo node JSON objects using `New-Port` helpers and GUIDs → serialize to `.dyn` JSON → write file. Modifying what a generated graph does means editing the embedded Python `Code` string inside the generator script.

### Python Scripts

Python files are embedded in Dynamo "Python Script" nodes or loaded as external scripts. Key patterns used throughout:

- **Revit API access**: `clr.AddReference('RevitAPI')` + `from Autodesk.Revit.DB import *`
- **Document context**: `DocumentManager.Instance.CurrentDBDocument`
- **Transaction management**: Always use `TransactionManager.Instance.EnsureInTransaction(doc)` / `ForceCloseTransaction()`
- **Unit conversion**: Revit stores lengths in internal units (feet); convert to/from mm with `UnitUtils.ConvertFromInternalUnits` / `ConvertToInternalUnits`
- **Geometry**: Bounding box and triangulation operations for Z-surface attachment (`optimized_import_top_solid_to_family_z_move.py` — 1,035 lines — is the most complex file)

### Dynamo Graph JSON (`.dyn`)

`.dyn` files are JSON. Key top-level fields: `Nodes` (array of node objects), `Connectors` (wiring), `View`. Each node has `ConcreteType`, `Id` (GUID), `Inputs`, `Outputs`. Python Script nodes store their code in a `Code` field. When a PowerShell generator produces a graph, it injects Python source directly into this `Code` field.

## Development Workflow

To modify a workflow:
1. Edit the Python `.py` file for the module
2. If the graph was generated by a PowerShell script, re-run that script to regenerate the `.dyn` (or paste the updated code directly into the Dynamo Python node)
3. Test by running the `.dyn` in Dynamo inside Revit with a test model

To add a new generated graph, create a new PowerShell script in `scripts/` following the existing generator pattern.

## User Learning Preference

The user is a coding beginner and wants to learn while changes are being made.

When creating or modifying code, explain the work in a beginner-friendly way:

1. State what problem was observed.
2. Explain the cause in plain language.
3. Name the programming concept the user should learn from the issue.
4. Describe what code was changed and why that change fixes the problem.
5. Include clickable file links with line numbers for the important edited code.
6. Show the important edited code snippets in the explanation, and place the related file/line link next to each snippet.
7. Explain what the edited code does, not only that it was changed.
8. When useful, mention how the user could recognize or debug a similar issue next time.

Prefer this teaching-oriented explanation style for both new code and bug fixes, including in future Revit add-ins created in this repository.
