## 슬라이드 1. BIM 기초 강의

안녕하세요. 오늘은 Revit을 처음 접하는 분들을 대상으로 BIM 기초와 Revit 실무 흐름을 같이 살펴보겠습니다.

이번 강의의 목표는 단순히 버튼 위치를 외우는 것이 아닙니다. Revit에서 모델을 만들고, 정보를 입력하고, 협업하고, 최종 납품 파일까지 정리하는 전체 흐름을 이해하는 것입니다.

건축, 역사, 전기공종 모델링 업무에서 Revit이 어떤 역할을 하는지 큰 그림부터 시작하겠습니다.

## 슬라이드 2. 목차 1

먼저 오늘 다룰 큰 목차를 보겠습니다.

앞부분에서는 BIM 개념과 Revit의 역할을 다룹니다. 3D 시각화, 간섭 검토, 수량산출, 속성정보 입력처럼 실무에서 Revit을 쓰는 이유를 먼저 이해합니다.

그다음 Revit 인터페이스를 보고, 리본 메뉴, 프로젝트 브라우저, 속성 패널 같은 기본 화면 구조를 익히겠습니다. 이후에는 공동작업, 패밀리 작성, 링크 파일 불러오기까지 순서대로 진행합니다.

## 슬라이드 3. 목차 2

뒤쪽 목차는 실무 결과물과 직접 연결되는 내용입니다.

속성정보 입력, 물량 산출, 납품 파일 추출, 간섭검토까지 이어집니다. 모델을 만드는 것에서 끝나는 것이 아니라, 그 모델이 수량표, NWC, IFC, 간섭검토 보고서 같은 결과물로 어떻게 이어지는지를 보는 흐름입니다.

오늘 강의는 각 기능을 따로 외우기보다, 실제 프로젝트 제출 절차와 연결해서 이해하는 데 초점을 맞추겠습니다.

## 슬라이드 4. BIM 개념과 Revit의 역할

첫 번째 파트는 BIM 개념과 Revit의 역할입니다.

Revit은 단순히 3D 모델을 그리는 프로그램이 아닙니다. 객체에 정보가 들어가고, 그 정보가 도면, 수량, 검토, 납품 파일로 연결되는 BIM 도구입니다.

이 파트에서는 왜 Revit을 쓰는지, 그리고 Revit으로 만든 모델이 실무에서 어떤 결과물로 이어지는지를 먼저 정리하겠습니다.

## 슬라이드 5. BIM 개념 핵심

BIM의 실무 가치는 크게 다섯 가지로 볼 수 있습니다.

첫째, 3D 시각화와 도면화입니다. 모델을 만들면 평면, 단면, 입면 같은 도면이 모델과 연결되어 관리됩니다.

둘째, 간섭 검토입니다. 전기, 건축, 기계 등 여러 분야 모델을 겹쳐서 충돌 여부를 확인할 수 있습니다.

셋째, 정보 활용입니다. 객체에 속성정보를 입력하면 수량산출이나 태그, IFC 정보로 활용할 수 있습니다.

넷째, 협업입니다. Worksharing을 통해 여러 작업자가 같은 프로젝트를 나누어 작업할 수 있습니다.

마지막으로 최종 성과물입니다. 프로젝트 기준에 따라 RVT, NWC, NWD, IFC 같은 파일로 제출됩니다.

## 슬라이드 6. Revit 영어버전으로 변경

실무에서는 Revit을 영어 버전으로 사용하는 경우가 많습니다.

그 이유는 실무 자료, 플러그인, 오류 메시지, 검색 결과가 영어 기준인 경우가 많기 때문입니다. 그래서 같은 기능을 찾더라도 영어 메뉴명을 알고 있으면 훨씬 빠르게 대응할 수 있습니다.

변경은 Revit 파일 자체를 바꾸는 것이 아니라, 바로가기 속성에서 언어 코드 부분을 `KOR`에서 `enu`로 바꾸는 방식입니다. 경로를 수정할 때는 원래 값을 확인하고 변경해야 합니다.

## 슬라이드 7. Revit Interface

두 번째 파트는 Revit 인터페이스입니다.

처음 Revit 화면을 보면 복잡해 보일 수 있지만, 큰 덩어리로 나누어 보면 이해하기 쉽습니다. 리본 메뉴, 속성 패널, 프로젝트 브라우저, 작업 뷰가 각각 어떤 역할을 하는지만 먼저 잡으면 됩니다.

이 파트에서는 기능을 모두 외우는 것이 아니라, 필요한 기능을 어디서 찾아야 하는지 감을 잡는 데 집중하겠습니다.

## 슬라이드 8. Revit 화면 전체 구조

Revit 화면은 크게 리본 메뉴, 속성 패널, 프로젝트 브라우저, 작업 화면으로 나눌 수 있습니다.

리본 메뉴는 명령을 실행하는 곳입니다. 속성 패널은 현재 선택한 객체나 뷰의 설정을 확인하는 곳입니다. 프로젝트 브라우저는 도면, 뷰, 시트, 패밀리를 관리하는 곳입니다.

여기서 중요한 주의사항이 있습니다. 기본 도면, 예를 들어 기본 Floor Plan은 바로 수정하지 말고 복사해서 작업뷰로 사용하는 습관을 들이는 것이 좋습니다. 그래야 원본 뷰를 보호하면서 작업할 수 있습니다.

## 슬라이드 9. 리본 메뉴 이해하기

리본 메뉴와 화면 하단 옵션은 뷰 표시와 선택 방식에 큰 영향을 줍니다.

Detail Level은 객체를 어느 정도 상세하게 보여줄지 결정하고, Visual Style은 음영이나 선 표시 방식을 바꿉니다. 뷰 자르기, 임시 숨기기, 링크 선택, Pin 객체 선택 같은 옵션도 작업 효율에 직접 영향을 줍니다.

초보자는 객체가 사라졌다고 생각하기 쉽지만, 실제로는 숨김, 분리, 뷰 범위, 필터 설정 때문에 보이지 않는 경우가 많습니다. 그래서 화면 표시 옵션을 먼저 확인하는 습관이 중요합니다.

## 슬라이드 10. 공동작업

세 번째 파트는 공동작업, 즉 Worksharing입니다.

Revit 프로젝트에서 여러 사람이 동시에 작업하려면 중앙 파일과 로컬 파일 개념을 반드시 이해해야 합니다. 여기서는 중앙 파일을 만들고, VBS로 로컬 파일을 생성하고, Workset과 Synchronize를 관리하는 흐름을 보겠습니다.

공동작업은 파일을 만드는 기술보다 운영 습관이 더 중요합니다. 특히 중앙 파일을 직접 열지 않는 원칙을 꼭 기억해야 합니다.

## 슬라이드 11. Worksharing 설정 기본

Central file은 여러 작업자가 공유하는 기준 모델입니다. Local file은 각 작업자가 실제로 작업하는 개인 작업 파일입니다.

Workset은 작업 영역을 나누는 기준입니다. 공종별, 작업자별로 영역을 나누면 누가 어느 부분을 작업하는지 관리하기 쉬워집니다.

Synchronize는 내 변경사항을 중앙 파일에 반영하고, 다른 작업자의 변경사항을 가져오는 과정입니다. 중앙 파일 직접 작업 금지, 동일한 드라이브 경로 사용, 작업 전 로컬 파일 확인, 작업 후 동기화 습관이 핵심입니다.

## 슬라이드 12. Central File 생성

중앙 파일을 만들 때는 먼저 드라이브 경로를 정확히 잡아야 합니다. 모든 작업자가 같은 경로로 접근할 수 있어야 하기 때문입니다.

그다음 파일을 열고 Central File로 저장합니다. 저장 후에는 다시 한 번 저장해서 설정이 정상적으로 반영되었는지 확인합니다.

중요한 점은 중앙 파일명이나 경로가 바뀌면 Central File을 다시 생성해야 할 수 있다는 것입니다. 경로와 파일명은 처음부터 기준을 정해 관리하는 것이 좋습니다.

## 슬라이드 13. Central File 생성 확인

중앙 파일이 정상적으로 만들어졌는지는 Sync 아이콘이 활성화되는지로 확인할 수 있습니다.

이 아이콘이 활성화되어 있으면 현재 파일이 공동작업 구조로 설정되었다고 볼 수 있습니다.

하지만 중앙 파일을 만든 뒤에도 실제 작업은 중앙 파일에서 하지 않습니다. 반드시 VBS 파일이나 별도 절차를 통해 로컬 파일을 만들어서 작업해야 합니다. 중앙 파일을 직접 열어 작업하면 파일 잠김이나 동기화 충돌이 생길 수 있습니다.

## 슬라이드 14. Central File 재생성

중앙 파일의 경로나 이름이 변경되었거나, 기존 중앙 파일을 다시 기준 파일로 잡아야 할 때는 재생성 절차가 필요합니다.

수정할 Central File을 Detached file로 열고, 파일명과 경로를 수정한 뒤 저장합니다. 이후 Central File로 다시 설정하고 저장을 한 번 더 진행합니다.

이 과정은 공동작업 기준을 다시 잡는 작업이므로, 작업자들이 기존 로컬 파일을 계속 사용해도 되는지 반드시 확인해야 합니다.

## 슬라이드 15. VBS 파일 생성

VBS 파일은 작업자가 로컬 파일을 안정적으로 만들 수 있도록 도와주는 실행 파일입니다.

파일명은 Central File 이름과 동일하게 맞추면 관리하기 쉽습니다. 그리고 로컬 파일이 생성될 드라이브 경로를 정확히 설정해야 합니다.

경로가 잘못되면 작업자마다 다른 위치에 로컬 파일이 생기거나, 파일을 찾지 못하는 문제가 발생할 수 있습니다. 그래서 VBS는 만들고 끝내는 것이 아니라 실제 실행 테스트까지 해야 합니다.

## 슬라이드 16. Workset 생성

Workset은 공종이나 작업자 기준으로 작업 영역을 나누는 기능입니다.

예를 들어 `EL_lhj`처럼 공종과 작업자 이름을 조합하면 누가 어떤 영역을 담당하는지 파악하기 쉽습니다.

Editable은 보통 `NO`로 두어 소유권을 독점하지 않도록 합니다. Opened는 `YES`로 두면 해당 Workset 객체들이 화면에 보입니다. 공동작업에서는 내가 편하다고 모든 것을 소유하는 것보다, 필요한 범위만 가져오고 공유 상태를 유지하는 것이 중요합니다.

## 슬라이드 17. Synchronize

Synchronize는 공동작업에서 가장 중요한 습관 중 하나입니다.

내가 수정한 내용을 중앙 파일에 반영하고, 다른 작업자가 수정한 내용도 내 로컬 파일로 가져옵니다. 프로젝트 규모에 따라 다르지만 보통 30분에서 2시간 정도 주기로 동기화하는 것을 권장합니다.

동기화 전에는 권한 포기, 즉 Relinquish를 확인해야 합니다. 내가 계속 권한을 잡고 있으면 다른 작업자가 같은 요소를 수정하지 못할 수 있습니다.

## 슬라이드 18. 패밀리 작성

네 번째 파트는 패밀리 작성입니다.

Revit에서 패밀리는 반복적으로 사용하는 객체의 기본 단위입니다. 전등, 분전반, 특고압반처럼 프로젝트에 배치되는 요소들이 패밀리로 관리됩니다.

이 파트에서는 패밀리를 어떤 순서로 만들고, Instance Parameter와 Type Parameter를 어떻게 구분하는지 살펴보겠습니다.

## 슬라이드 19. 패밀리 작성 기본 흐름

패밀리 작성은 템플릿 선택, 카테고리 설정, 기준선 설정, 형상 작성, 파라미터와 재질 입력 순서로 진행하면 이해하기 쉽습니다.

초보자는 형상부터 만들고 싶어 하지만, 기준면과 기준선을 먼저 잡아야 나중에 치수와 파라미터가 안정적으로 작동합니다.

Instance Parameter는 배치된 객체마다 달라질 수 있는 값이고, Type Parameter는 같은 타입 전체에 공통으로 적용되는 값입니다. 만든 뒤에는 반드시 프로젝트에 로드해서 실제 뷰에서 테스트해야 합니다.

## 슬라이드 20. 패밀리 작성 실습

이번 실습에서는 전등, 저압반, 특고압반 같은 전기 객체를 예로 듭니다.

전등은 A-TYPE, D-TYPE처럼 객체에 따라 기준선이 달라질 수 있습니다. 저압반은 Name plate나 Door handle 같은 부속 요소를 넣고, 타입별로 켜고 끄는 설정을 만들 수 있습니다.

패밀리명과 타입명은 나중에 물량산출과 속성정보 관리에 영향을 줍니다. 그래서 처음부터 프로젝트 기준에 맞게 이름 규칙을 잡는 것이 중요합니다.

## 슬라이드 21. CAD Import

패밀리 작성 실습의 첫 단계는 CAD 도면을 가져오는 것입니다.

CAD는 형상을 그릴 때 참고 기준으로 사용할 수 있습니다. 다만 CAD를 그대로 믿고 모델링하기보다, 기준선과 치수 기준을 확인하면서 필요한 부분만 참고하는 것이 좋습니다.

불필요한 CAD 선이 많으면 패밀리 파일이 무거워질 수 있으므로, 가져오기 전에 CAD 파일을 정리하는 습관도 중요합니다.

## 슬라이드 22. Create

이 단계에서는 실제 형상을 작성합니다.

Create 도구를 사용해 기본 형상을 만들고, 기준면과 치수에 맞춰 위치를 잡습니다. 단순히 눈대중으로 그리는 것이 아니라 기준선과 파라미터에 연결해야 나중에 크기 변경이 안정적으로 됩니다.

패밀리는 처음부터 완벽하게 만들기보다, 기본 형상을 만들고 프로젝트에 로드해서 확인한 뒤 수정하는 방식으로 진행하는 것이 좋습니다.

## 슬라이드 23. 패밀리 재질입력

패밀리에는 형상뿐 아니라 재질 정보도 입력해야 합니다.

재질은 화면 표현뿐 아니라 프로젝트 기준에 따라 속성정보와 납품 데이터에도 영향을 줄 수 있습니다. 금속, 플라스틱, 도장면처럼 구분이 필요한 항목은 기준에 맞게 입력합니다.

재질 입력 후에는 프로젝트에 로드해서 실제 뷰에서 표시가 의도대로 되는지 확인합니다.

## 슬라이드 24. 링크 파일 불러오기

다섯 번째 파트는 Revit과 CAD 파일을 링크로 불러오는 방법입니다.

링크는 파일을 현재 모델 안에 복사해 넣는 것이 아니라 외부 파일을 참조하는 방식입니다. 따라서 원본 관리와 좌표 기준이 중요합니다.

이 파트에서는 Revit 링크와 CAD 링크의 차이, 그리고 좌표 기준을 어떻게 선택해야 하는지 살펴보겠습니다.

## 슬라이드 25. Revit 링크와 CAD 링크 불러오기

Revit 링크는 다른 분야의 모델을 참조할 때 주로 사용하고, CAD 링크는 2D 도면 기준을 참조할 때 많이 사용합니다.

CAD 파일은 가능하면 WBLOCK으로 정리해서 필요한 도면만 링크하는 것이 좋습니다. 파일명도 위치와 용도를 알 수 있게 정리해야 나중에 관리가 쉽습니다.

좌표 기준은 프로젝트 상황에 따라 다릅니다. Internal Origin to Internal Origin은 내부 원점 기준이고, By Shared Coordinates는 공유 좌표 기준입니다. Project Base Point to Project Base Point는 프로젝트 내부 기준점이 합의되어 있을 때 사용합니다. 좌표를 잘못 선택하면 모델 위치가 크게 어긋날 수 있습니다.

## 슬라이드 26. 트레이 작성

여섯 번째 파트는 케이블 트레이 작성입니다.

트레이는 단순 선이 아니라 폭, 높이, 피팅, 연결 상태가 모두 중요한 모델 요소입니다. 전기 모델링에서 물량산출과 간섭검토에 직접 연결됩니다.

다음 슬라이드에서 작성 순서와 주의사항을 보겠습니다.

## 슬라이드 27. 트레이 작성 순서

트레이는 시스템 탭에서 트레이를 선택하고, 폭과 높이 규격을 먼저 지정한 뒤 경로를 따라 작성합니다.

건축전기 케이블 작성 시에는 무리하게 드래그하면 경사가 생길 수 있으므로 주의해야 합니다. 작성 후에는 피팅류가 정상적으로 연결되었는지 Analyze 상태를 확인합니다.

트레이와 피팅류의 패밀리명, 타입명도 기준에 맞게 관리해야 합니다. 이름 기준이 흔들리면 수량산출과 납품 데이터 정리에서 문제가 생깁니다.

## 슬라이드 28. 가시성 설정

일곱 번째 파트는 가시성 설정입니다.

모델이 많아질수록 필요한 객체를 잘 보이게 하고, 불필요한 객체를 숨기는 설정이 중요해집니다. 여기서는 Filter 생성과 View Template 생성이 핵심입니다.

가시성 설정은 보기 좋게 만드는 기능이 아니라, 검토와 작업 효율을 높이는 기본 설정이라고 이해하면 됩니다.

## 슬라이드 29. 가시성 설정 주요 기능

가시성 설정은 `VV` 또는 Visibility/Graphics에서 많이 다룹니다.

Filter는 원하는 객체를 색상이나 선 스타일로 구분할 때 사용합니다. View Template은 여러 뷰에 같은 설정을 반복 적용할 수 있게 해줍니다.

View Range는 평면뷰에서 어떤 높이 범위의 객체가 보이는지를 결정합니다. 객체가 안 보일 때는 삭제되었다고 생각하기보다, 필터와 뷰 범위부터 확인하는 습관이 필요합니다.

## 슬라이드 30. 속성정보 입력

여덟 번째 파트는 속성정보 입력입니다.

BIM 모델은 형상만 있는 3D 모델이 아닙니다. 객체마다 필요한 데이터가 들어가야 물량산출, 태그, IFC 변환, 납품 자료로 활용할 수 있습니다.

이제 객체 정보를 관리하는 기본 단위인 Parameter를 보겠습니다.

## 슬라이드 31. 객체의 데이터 관리

Parameter는 객체 정보를 관리하는 핵심입니다.

프로젝트 파라미터는 프로젝트 내에서 적용되지만 패밀리 자체로 전달되지는 않습니다. 패밀리 파라미터는 개별 RFA 파일 안에서 정의됩니다. 공유 파라미터는 외부 텍스트 파일에 저장되어 여러 파일에서 공유할 수 있고, Schedule이나 태그에 활용할 때 중요합니다.

시스템 파라미터는 Revit에 기본 내장된 값이라 삭제할 수 없습니다. 어떤 파라미터를 써야 하는지는 나중에 데이터를 어디에 쓸 것인지에 따라 결정해야 합니다.

## 슬라이드 32. 프로젝트 파라미터 입력

물량 산출의 첫 단계는 필요한 프로젝트 파라미터를 입력하는 것입니다.

수량은 모델 형상만으로 항상 충분히 나오지 않습니다. 구역, 공종, 시스템명, 납품 기준 같은 속성값이 있어야 원하는 기준으로 집계할 수 있습니다.

그래서 수량산출 전에 어떤 데이터를 입력해야 하는지 먼저 정리하는 것이 중요합니다.

## 슬라이드 33. 물량 산출

아홉 번째 파트는 물량 산출입니다.

여기서는 Schedule 작성과 케이블 길이 추출을 중심으로 봅니다. Schedule은 Revit 안의 객체 정보를 표 형태로 확인하는 기능이고, 케이블 길이 추출은 모델의 경로 정보와 연결 상태가 정확해야 가능합니다.

물량 산출은 마지막에 버튼 하나로 끝나는 작업이 아니라, 앞 단계의 모델링과 속성정보 입력 품질에 영향을 받습니다.

## 슬라이드 34. Schedule 작성과 케이블 길이 추출

Schedule 작성에서는 먼저 산출할 카테고리를 정확히 선택해야 합니다.

트레이라면 length, width, height 같은 필요한 필드를 추가하고 값이 제대로 들어오는지 확인합니다. 값이 비어 있거나 형식이 다르면 산출 결과도 틀어집니다.

케이블 길이 추출은 경로와 연결 상태가 중요합니다. 모델 검토를 먼저 하고 산출하는 순서가 안정적입니다.

## 슬라이드 35. 납품 파일 추출

열 번째 파트는 납품 파일 추출입니다.

Revit 모델만 제출하는 것이 아니라, 프로젝트 기준에 따라 NWC, NWF, NWD, IFC 같은 여러 형식으로 변환하고 정리해야 합니다.

이 파트에서는 각 파일 형식의 역할과 추출 전 설정값을 살펴보겠습니다.

## 슬라이드 36. Navisworks Exporters 설치

NWC 파일을 추출하려면 Navisworks Exporters가 설치되어 있어야 합니다.

중요한 점은 설치된 Revit 버전에 맞는 Exporter를 설치해야 한다는 것입니다. 버전이 맞지 않으면 Revit에서 Export 메뉴가 보이지 않거나 추출 오류가 날 수 있습니다.

설치 후에는 Revit에서 실제 Export 메뉴가 정상적으로 나타나는지 확인해야 합니다.

## 슬라이드 37. NWC 추출 시 Revit 설정값

NWC는 Navisworks에서 열 수 있는 캐시 파일입니다. NWC 안에서는 객체를 직접 수정할 수 없고, 원본 수정은 Revit에서 해야 합니다.

추출 전에는 Detail Level과 Visibility 설정을 확인합니다. 예를 들어 Cable Tray는 medium, 나머지 객체는 fine처럼 프로젝트 기준에 맞게 설정할 수 있습니다.

또한 `VV`에서 Model Categories 외의 불필요한 탭을 비활성화하여 추출 범위를 정리합니다.

## 슬라이드 38. NWC 추출 시 Navisworks 설정값

NWC 추출 시에는 Navisworks Tool의 Settings도 확인해야 합니다.

Convert element properties, coordinates, links 같은 항목은 공동작업과 검토에 영향을 줄 수 있습니다. 프로젝트 기준에 따라 설정값은 달라질 수 있으므로, 화면의 예시를 기준으로 현재 프로젝트 기준과 비교해 확인합니다.

설정값은 한 번 맞췄다고 끝나는 것이 아니라, 납품 기준이 바뀌면 다시 확인해야 합니다.

## 슬라이드 39. NWC, NWF, NWD

NWC, NWF, NWD는 역할이 다릅니다.

NWC는 Revit 등에서 추출한 캐시 파일입니다. NWF는 여러 NWC를 링크로 불러와 검토하는 작업 파일입니다. NWC 위치가 바뀌면 NWF에서 경로를 다시 잡아야 할 수 있습니다.

NWD는 납품 시 사용하는 읽기 전용 통합 파일입니다. 검토 결과를 포함해서 전달하기 좋기 때문에 제출용으로 많이 사용됩니다.

## 슬라이드 40. IFC 개념

IFC는 Industry Foundation Classes의 약자로, BIM 표준 교환 포맷입니다.

다른 소프트웨어와 모델 정보를 주고받기 위해 사용합니다. 단순 형상만 넘기는 것이 아니라 속성정보와 분류 정보도 함께 전달할 수 있습니다.

다만 Revit의 모든 정보가 그대로 넘어가는 것은 아닙니다. 그래서 IFC Class Mapping과 추출 설정을 반드시 확인해야 합니다.

## 슬라이드 41. IFC Class Mapping

IFC Class Mapping은 Revit Category를 IFC 분류 체계에 맞게 연결하는 과정입니다.

예를 들어 Cable Trays는 IFC에서 Cable Carrier Segment 계열로, Cable Tray Fittings는 Cable Carrier Fitting 계열로 매핑됩니다.

IfcExportAs 값이 잘못되면 객체가 엉뚱한 IFC 분류로 나갈 수 있습니다. 납품 기준에서 요구하는 IFC 클래스가 무엇인지 확인하고, 매핑 파일이나 파라미터를 관리해야 합니다.

## 슬라이드 42. IFC 추출 설정값

IFC 추출 설정값은 프로젝트 기준에 따라 달라질 수 있습니다.

좌표, 속성정보 포함 여부, 링크 포함 여부, IFC 버전 같은 항목을 확인해야 합니다. 특히 IFC2x3과 IFC4는 요구 기준이 다를 수 있으므로 납품 기준서를 먼저 확인합니다.

추출 후에는 IFC 뷰어에서 열어 누락, 위치 오류, 분류 오류가 없는지 반드시 검토합니다.

## 슬라이드 43. 파일정리

납품 전 파일정리는 매우 중요합니다.

불필요한 3D View, 작업뷰, 링크, 필터, 미사용 패밀리가 남아 있으면 파일이 복잡해지고 검토자가 혼동할 수 있습니다.

패밀리명이 기준과 맞는지, 중복된 이름이 없는지, 제출에 불필요한 요소가 남아 있지 않은지 확인합니다. 파일정리는 단순 청소가 아니라 납품 품질 관리입니다.

## 슬라이드 44. 간섭검토

마지막 주요 파트는 간섭검토입니다.

간섭검토는 모델을 만든 뒤 문제가 없는지 확인하는 검토 단계입니다. 특히 전기, 건축, 기계, 토목 모델이 함께 있을 때는 충돌 조건과 검토 범위를 정확히 설정해야 합니다.

이 파트에서는 Navisworks에서 Set을 만들고, 조건을 설정하고, Report를 내보내는 흐름을 보겠습니다.

## 슬라이드 45. Set 설정과 Save Selection

간섭검토를 하려면 먼저 검토할 객체 그룹을 Set으로 관리합니다.

Selection Tree에서 객체나 파일을 선택하고, Sets에서 오른쪽 클릭 후 Save Selection을 선택하면 현재 선택된 객체를 Set으로 저장할 수 있습니다.

Save Selection은 선택한 객체가 고정되어 있을 때 유용합니다. 변경이 거의 없는 기준 모델이나 특정 파일을 계속 검토할 때 사용하기 좋습니다.

## 슬라이드 46. Save Search

Save Search는 조건으로 객체를 찾는 방식입니다.

Find Item에서 검색 조건을 입력하고, File All 또는 특정 File을 선택해 검색 범위를 정합니다. 특정 File을 선택하면 그 파일 안에서만 조건을 찾고, 선택하지 않으면 전체 파일에서 검색할 수 있습니다.

Save Search는 모델이 갱신되어도 조건에 맞는 객체를 다시 찾을 수 있어 반복 검토에 유리합니다. 다만 조건을 잘못 만들면 원하지 않는 객체까지 포함될 수 있으므로 결과 확인이 필요합니다.

## 슬라이드 47. Rule 설정

Rule 설정은 조건부 간섭을 걸러내기 위한 단계입니다.

예를 들어 실제로 문제가 되지 않는 같은 파일 내부 객체 간 충돌이나, 특정 조건의 간섭을 제외할 수 있습니다. 이렇게 하면 검토자가 중요한 문제에 집중할 수 있습니다.

하지만 Rule을 너무 많이 적용하면 중요한 간섭까지 빠질 수 있습니다. 기준을 명확히 하고, 어떤 간섭을 제외했는지 기록하는 것이 좋습니다.

## 슬라이드 48. 간섭 항목 조건 변경

이 슬라이드는 간섭 항목 조건을 바꾸면서 결과가 어떻게 달라지는지 보여주는 단계입니다.

초보자에게는 조건을 조금 바꾸면 간섭 개수와 결과가 크게 달라진다는 점을 직접 보여주는 것이 좋습니다.

중요한 것은 단순히 간섭 개수만 보는 것이 아닙니다. 실제 조치가 필요한 간섭인지, 기준에 따라 제외 가능한 간섭인지 판단해야 합니다.

## 슬라이드 49. 간섭 결과 Report

마지막으로 간섭 결과를 Report로 정리합니다.

Report는 단순 캡처가 아니라, 어떤 간섭을 확인했고 현재 상태가 무엇인지 기록하는 산출물입니다. Contents와 Status를 적절히 선택해야 검토자와 작업자가 같은 기준으로 결과를 볼 수 있습니다.

보고서에는 문제 위치, 상태, 담당자 판단이 명확하게 남아야 이후 조치와 재검토가 가능합니다.

## 슬라이드 50. 마무리

오늘은 Revit 초급자가 BIM 모델링 업무의 전체 흐름을 이해할 수 있도록 주요 내용을 살펴봤습니다.

BIM 개념, Revit 화면, 공동작업, 패밀리, 링크, 트레이, 가시성 설정, 속성정보, 물량산출, 납품 파일, 간섭검토까지 모두 서로 연결되어 있습니다.

실무에서는 한 번에 모든 기능을 완벽하게 하려고 하기보다, 기준을 정확히 잡고 반복해서 검토하는 습관이 중요합니다. 특히 패밀리명 기준과 속성정보 기준은 프로젝트 전체 품질에 영향을 주므로 처음부터 일관되게 관리해야 합니다.

이상으로 BIM 기초 강의를 마치겠습니다. 감사합니다.
