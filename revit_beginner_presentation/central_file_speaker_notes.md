# Revit BIM 초급 강의 스피치 노트

대상 파일: `bim_beginner_training_v17_최종본.pptx`  
용도: 발표자 화면 또는 강의 전 리허설용 메모

## 슬라이드 1. BIM 기초 강의

- 오늘 강의는 Revit을 처음 접하는 분들이 BIM 업무의 전체 흐름을 이해하는 것을 목표로 합니다.
- 단순히 버튼 위치를 외우는 수업이 아니라, 모델을 만들고 관리하고 납품하는 기본 흐름을 잡는 시간이라고 안내해 주세요.
- 건축·역사·전기공종 모델링 업무에서 Revit이 어떤 역할을 하는지 큰 그림부터 시작합니다.

## 슬라이드 2. 목차 1

- 목차의 앞부분은 Revit을 사용하기 위한 기본 개념과 화면 조작, 공동작업, 패밀리 작성까지의 흐름입니다.
- 초보자는 기능을 개별적으로 외우기 쉬우므로, 각 항목이 실제 프로젝트 업무 순서와 연결된다는 점을 먼저 설명해 주세요.
- 오늘은 전체 과정을 훑고, 각 단계별로 실무에서 자주 실수하는 포인트를 같이 짚어보겠다고 안내합니다.

## 슬라이드 3. 목차 2

- 후반부는 모델을 만든 뒤 데이터를 입력하고, 물량을 산출하고, 납품 파일을 정리하고, 간섭검토까지 이어지는 과정입니다.
- 이 부분은 단순 모델링 이후 실제 BIM 성과품으로 연결되는 단계라고 설명해 주세요.
- 앞부분이 Revit 사용 준비라면, 뒷부분은 프로젝트 제출과 검토에 필요한 실무 절차입니다.

## 슬라이드 4. BIM 개념과 Revit의 역할

- 첫 번째 파트는 BIM 개념과 Revit의 역할입니다.
- Revit을 3D 모델링 프로그램으로만 보면 기능 이해가 좁아지므로, 정보가 담긴 모델을 만들고 활용하는 도구라는 점을 강조해 주세요.
- 이 섹션에서는 왜 Revit을 쓰는지, 어떤 결과물로 이어지는지를 먼저 잡아줍니다.

## 슬라이드 5. BIM 개념 핵심

- BIM의 핵심 가치는 3D 시각화, 간섭 검토, 정보 활용, 협업 가능성으로 정리할 수 있습니다.
- 3D 모델을 만들면 도면화, 수량산출, 속성정보 입력까지 이어진다는 점을 예시로 설명해 주세요.
- 최종 성과물이 RVT, NWC, NWD, IFC 등으로 확장된다는 점을 말하며 후반부 납품 파트와 연결합니다.

## 슬라이드 6. Revit 영어버전으로 변경

- Revit을 영어 버전으로 바꾸는 이유는 실무 자료, 플러그인, 오류 메시지, 검색 결과가 영어 기준인 경우가 많기 때문입니다.
- 바로가기 속성에서 언어 코드를 수정하는 방식이므로, 파일 자체를 바꾸는 것이 아니라 실행 옵션을 바꾸는 절차라고 설명해 주세요.
- 초보자는 경로와 띄어쓰기를 잘못 수정하기 쉬우니 변경 전 원본 값을 확인하는 습관을 강조합니다.

## 슬라이드 7. Revit Interface

- 두 번째 파트는 Revit 인터페이스입니다.
- 처음에는 화면이 복잡해 보이지만, 리본 메뉴, 속성창, 프로젝트 브라우저, 작업 뷰로 나누어 보면 이해가 쉬워집니다.
- 이 섹션에서는 기능을 외우기보다 어디에서 찾아야 하는지 감을 잡는 데 초점을 둡니다.

## 슬라이드 8. Revit 화면 전체 구조

- 화면을 큰 덩어리로 나누어 설명해 주세요. 위쪽은 명령을 실행하는 리본 메뉴, 좌측은 속성, 우측 또는 좌측 패널은 프로젝트 브라우저입니다.
- 프로젝트 브라우저에서는 도면, 뷰, 시트, 패밀리를 관리한다는 점을 강조합니다.
- 도면 작업 시 기본 뷰를 직접 수정하지 말고 복사해서 사용하는 습관을 초반부터 안내해 주세요.

## 슬라이드 9. 리본 메뉴 이해하기

- 리본 메뉴와 화면 하단의 뷰 제어 기능을 함께 설명하는 슬라이드입니다.
- Detail Level, Visual Style, 뷰 자르기, 임시 숨기기 등은 화면 표시와 검토 효율에 직접 영향을 줍니다.
- 초보자는 객체가 사라졌다고 생각하기 쉬우므로, 숨김/분리, 링크 선택, 핀 객체 선택 같은 옵션을 꼭 확인하도록 안내합니다.

## 슬라이드 10. 공동작업

- 세 번째 파트는 공동작업입니다. Revit 프로젝트에서 여러 사람이 동시에 작업하려면 central file과 local file 개념이 반드시 필요합니다.
- 이 섹션에서는 중앙 파일을 만드는 것뿐 아니라, VBS로 로컬 파일을 생성하고 동기화하는 운영 습관까지 다룹니다.
- 중앙 파일을 직접 열어 작업하지 않는다는 원칙을 먼저 강조해 주세요.

## 슬라이드 11. 공동작업 Worksharing 설정 기본

- Central file은 팀이 공유하는 기준 모델이고, local file은 각 작업자가 실제로 작업하는 개인 작업 파일입니다.
- Workset은 작업 영역을 나누는 기준이고, Synchronize는 각자의 변경사항을 중앙 파일에 반영하는 과정입니다.
- 주의사항에서 가장 중요한 것은 중앙 파일 직접 작업 금지, 동일한 드라이브 경로 사용, 작업 전 로컬 파일 확인입니다.

## 슬라이드 12. Central File 생성

- Central File 생성 절차의 첫 부분입니다. 먼저 드라이브 경로를 정확히 설정하고 파일을 열어야 합니다.
- 공유 경로는 팀원 모두가 동일하게 접근 가능한 위치여야 한다는 점을 설명해 주세요.
- 저장 전후로 파일이 중앙 파일로 설정되는 흐름을 확인하도록 안내합니다.

## 슬라이드 13. Central File 생성 확인

- Central File 생성 후에는 Sync 아이콘이 활성화되는지 확인합니다.
- 이 아이콘은 현재 파일이 공동작업 구조로 정상 설정되었는지 확인하는 중요한 기준입니다.
- 중앙 파일을 만든 다음에는 반드시 로컬 파일에서 작업해야 하며, 중앙 파일을 직접 열면 파일 잠김이나 동기화 충돌이 생길 수 있다고 강조합니다.

## 슬라이드 14. VBS 파일 생성

- VBS 파일은 로컬 파일을 안정적으로 생성하기 위한 보조 실행 파일이라고 설명합니다.
- 센트럴 파일명과 VBS 파일명을 동일하게 맞추면 작업자가 헷갈리지 않고 사용할 수 있습니다.
- 로컬 드라이브 경로 설정이 잘못되면 개인 작업 파일이 엉뚱한 위치에 생길 수 있으므로 주의해야 합니다.

## 슬라이드 15. Workset 생성

- Workset은 공종별 또는 작업자별로 작업 영역을 구분하기 위한 기준입니다.
- 예시처럼 공종명과 작업자 이름을 조합하면 누가 어떤 영역을 작업하는지 파악하기 쉽습니다.
- Editable은 NO, Opened는 YES로 두는 운영 방식을 설명하고, 권한을 독점하지 않는 것이 협업에 유리하다고 안내합니다.

## 슬라이드 16. Synchronize

- Synchronize는 공동작업에서 저장 이상의 의미를 갖습니다. 내 변경사항을 중앙 파일에 반영하고 다른 사람의 변경사항도 가져오는 과정입니다.
- 30분에서 2시간 정도의 주기로 동기화하는 습관을 권장한다고 설명해 주세요.
- 동기화 전에는 권한 포기 relinquish를 확인해야 다른 작업자가 같은 요소를 편집할 수 있습니다.

## 슬라이드 17. 패밀리 작성

- 네 번째 파트는 패밀리 작성입니다.
- Revit에서 패밀리는 반복적으로 사용하는 객체의 기본 단위라고 설명해 주세요.
- 이번 섹션에서는 패밀리 작성 순서와 파라미터를 어떻게 구분하는지에 초점을 둡니다.

## 슬라이드 18. 패밀리 작성 기본 흐름

- 패밀리 작성은 템플릿 선택, 기준면 설정, 형상 작성, 파라미터 입력 순서로 진행하면 이해하기 쉽습니다.
- 초보자는 형상부터 만들려고 하지만, 기준면과 기준선을 먼저 잡아야 나중에 치수와 파라미터가 안정적으로 작동합니다.
- Instance parameter와 Type parameter의 차이는 프로젝트에 로드한 뒤 테스트하면서 다시 확인시키면 좋습니다.

## 슬라이드 19. 패밀리 작성 실습

- 이 슬라이드는 실제 작성할 패밀리 예시를 보여주는 실습 안내입니다.
- 전등, 저압반, 특고압반처럼 객체마다 기준선과 기준점이 다를 수 있음을 설명해 주세요.
- 패밀리명과 타입명은 나중에 물량산출과 속성정보 관리에 영향을 주므로, 처음부터 규칙을 맞추는 것이 중요합니다.

## 슬라이드 20. 패밀리 작성 기본 흐름 정리

- 여기서는 재질 입력과 테스트까지 포함한 패밀리 작성 흐름을 다시 정리합니다.
- 형상이 완성된 뒤에는 재질, 파라미터, 타입별 치수가 제대로 적용되는지 확인해야 합니다.
- 프로젝트에 로드한 후 실제 뷰에서 보이는 방식까지 검토해야 실무에서 사용할 수 있습니다.

## 슬라이드 21. 링크 파일 불러오기

- 다섯 번째 파트는 Revit과 CAD 링크 파일 불러오기입니다.
- 링크는 외부 파일을 참조하는 방식이므로, 삽입과 다르게 원본 관리가 중요하다고 설명해 주세요.
- 좌표 기준을 잘못 선택하면 모델 위치가 틀어질 수 있으므로 이 섹션에서는 좌표 개념을 함께 강조합니다.

## 슬라이드 22. Revit 링크와 CAD 링크 불러오기

- Revit 링크와 CAD 링크는 목적이 다릅니다. Revit 링크는 모델 참조, CAD 링크는 2D 도면 기준으로 활용하는 경우가 많습니다.
- Auto - Center to Center는 편하지만 실무 좌표 관리에는 한계가 있으므로, 내부 원점과 공유 좌표 기준도 함께 설명합니다.
- CAD 파일은 wblock으로 정리하고, 파일명은 위치와 용도를 알 수 있게 지정하는 것이 좋습니다.

## 슬라이드 23. 트레이 작성

- 여섯 번째 파트는 트레이 작성입니다.
- 전기·통신 모델링에서 트레이는 경로와 연결 상태가 중요하므로, 단순히 그리는 것보다 시스템 연결을 확인하는 과정이 필요합니다.
- 다음 슬라이드에서는 작성 순서와 명칭 기준을 중심으로 설명합니다.

## 슬라이드 24. 트레이 작성 순서

- 트레이는 시스템 탭에서 작성하고, 폭과 높이 규격을 먼저 확인한 뒤 경로를 따라 작성합니다.
- 없는 규격은 추가해야 하며, 건축전기 케이블 작성 시 무리하게 드래그하면 경사가 생길 수 있다는 점을 주의시켜 주세요.
- 작성 후에는 연결 부속과 Analyze 상태를 확인하여 모델이 끊기지 않았는지 점검합니다.

## 슬라이드 25. 가시성 설정

- 일곱 번째 파트는 가시성 설정입니다.
- Filter와 View Template은 모델을 보기 좋게 만드는 기능이 아니라, 검토와 작업 효율을 높이는 기본 설정입니다.
- 객체가 많아질수록 보기 설정이 중요해지므로 초보자에게 반드시 습관화해야 합니다.

## 슬라이드 26. 가시성 설정 주요 기능

- Filter는 원하는 객체를 색상이나 선 스타일로 구별하는 데 사용합니다.
- View Template은 뷰 설정을 반복 적용할 수 있어 여러 도면의 표현을 통일하는 데 유용합니다.
- View Range는 평면도에서 어떤 높이 범위의 객체가 보이는지를 결정하므로, 객체가 안 보일 때 먼저 확인해야 합니다.

## 슬라이드 27. 속성정보 입력

- 여덟 번째 파트는 속성정보 입력입니다.
- BIM 모델은 형상만 있는 3D 모델이 아니라, 객체별 정보가 들어간 데이터 모델이라는 점을 강조해 주세요.
- 속성정보는 후속 물량산출, 납품, IFC 변환과 연결됩니다.

## 슬라이드 28. 객체의 데이터 관리

- Parameter는 객체 정보를 관리하는 핵심입니다.
- 프로젝트 파라미터와 공유 파라미터, 인스턴스와 타입 파라미터를 구분해서 설명해 주세요.
- 초보자에게는 어떤 값이 개별 객체마다 달라지는지, 어떤 값이 타입 전체에 공통으로 적용되는지를 예시로 설명하면 이해가 쉽습니다.

## 슬라이드 29. 프로젝트 파라미터 입력

- 아홉 번째 파트는 물량 산출의 첫 단계인 프로젝트 파라미터 입력입니다.
- 물량 산출은 모델이 정확히 작성되어 있고, 필요한 속성값이 입력되어 있어야 가능합니다.
- 이 슬라이드에서는 산출 전에 데이터 준비가 먼저라는 점을 강조합니다.

## 슬라이드 30. 물량 산출

- 물량 산출은 크게 Schedule 작성과 케이블 길이 추출로 나누어 설명합니다.
- Schedule은 Revit 내부 데이터를 표 형태로 확인하는 기능이고, 케이블 길이 추출은 모델 요소의 길이 정보를 활용하는 과정입니다.
- 두 작업 모두 모델링 품질과 속성정보 입력 상태에 영향을 받습니다.

## 슬라이드 31. Schedule 작성과 케이블 길이 추출

- Schedule 작성 시에는 먼저 산출 대상 카테고리를 정확히 선택해야 합니다.
- 트레이의 length, width, height처럼 필요한 필드를 추가하고 값이 제대로 들어오는지 확인합니다.
- 케이블 길이 추출은 실제 경로와 연결 상태가 정확해야 하므로, 모델 검토 후 산출하는 순서가 좋습니다.

## 슬라이드 32. 납품 파일 추출

- 열 번째 파트는 납품 파일 추출입니다.
- RVT 모델에서 끝나는 것이 아니라 NWC, NWF, NWD, IFC 등 목적별 파일로 변환해 제출하는 단계입니다.
- 각 파일 형식의 역할을 구분해서 설명하면 후속 슬라이드를 이해하기 쉽습니다.

## 슬라이드 33. Navisworks Exporters 설치

- NWC 추출을 위해서는 Navisworks Exporters 설치 상태를 먼저 확인해야 합니다.
- Revit 버전과 Exporter 버전이 맞지 않으면 메뉴가 보이지 않거나 추출 오류가 생길 수 있습니다.
- 설치 후에는 Revit에서 Export 메뉴가 정상적으로 나타나는지 확인해 주세요.

## 슬라이드 34. NWC 파일 추출

- NWC는 Navisworks에서 열 수 있는 캐시 파일이며, 파일 안에서 객체를 직접 수정할 수는 없습니다.
- Export 시 Detail Level과 좌표 기준이 결과에 영향을 주므로 설정값을 확인해야 합니다.
- 검토용 파일이므로 모델 수정은 Revit 원본에서 하고 다시 추출하는 흐름을 설명합니다.

## 슬라이드 35. Navisworks 설정값

- NWC 추출 설정에서는 Navisworks Tool의 Settings 값을 확인합니다.
- Convert element properties, coordinates, links 등 프로젝트 기준에 따라 설정이 달라질 수 있습니다.
- 수업에서는 현재 프로젝트 기준 설정을 보여주되, 설정값은 발주처나 프로젝트 기준에 따라 변경될 수 있다고 안내해 주세요.

## 슬라이드 36. NWC, NWF, NWD

- NWF는 NWC 파일들을 링크로 관리하는 작업 파일입니다.
- NWD는 검토 결과를 포함해 전달하기 좋은 통합 파일로 이해하면 쉽습니다.
- NWC, NWF, NWD의 관계를 원본 데이터, 작업 파일, 공유용 결과 파일의 흐름으로 설명해 주세요.

## 슬라이드 37. IFC 개념

- IFC는 BIM 표준 교환 포맷입니다.
- 다른 소프트웨어와 모델 정보를 주고받기 위해 사용하며, 형상뿐 아니라 일부 속성정보도 함께 전달할 수 있습니다.
- 다만 Revit의 모든 정보가 그대로 넘어가는 것은 아니므로 매핑과 검수가 필요하다고 설명합니다.

## 슬라이드 38. IFC Class Mapping

- IFC Class Mapping은 Revit Category를 IFC 분류 체계에 맞춰 연결하는 과정입니다.
- IfcExportAs 값이 잘못되면 객체가 엉뚱한 IFC 분류로 나갈 수 있습니다.
- 납품 기준에 맞는 IFC 클래스가 무엇인지 확인하고 매핑 파일 또는 파라미터를 관리해야 합니다.

## 슬라이드 39. IFC 추출 설정값

- IFC 추출 설정값은 프로젝트 기준에 따라 달라질 수 있습니다.
- 좌표, 속성정보, 링크 포함 여부, IFC 버전 등을 확인해야 합니다.
- 추출 후에는 반드시 IFC 뷰어에서 열어 누락, 위치 오류, 분류 오류가 없는지 검토합니다.

## 슬라이드 40. 파일정리

- 납품 전 파일정리는 실무에서 매우 중요합니다.
- 불필요한 3D view, 작업뷰, 임시 뷰가 남아 있으면 파일이 복잡해지고 검토자가 혼동할 수 있습니다.
- 최종 제출 전에는 필요한 뷰와 시트, 링크, 파일명, 폴더 구조를 기준에 맞게 정리합니다.

## 슬라이드 41. 간섭검토

- 마지막 주요 파트는 간섭검토입니다.
- 간섭검토는 모델을 만든 뒤 문제가 없는지 확인하는 품질 검토 단계입니다.
- 설정에 따라 결과가 크게 달라지므로, 조건을 정확히 잡는 것이 중요하다고 설명해 주세요.

## 슬라이드 42. 간섭세팅 Set 설정

- Set 설정에서는 어떤 객체 그룹끼리 간섭을 볼 것인지 정합니다.
- Select와 Search를 편의에 맞게 사용하되, 검토 범위가 너무 넓으면 불필요한 간섭이 많이 나올 수 있습니다.
- 처음에는 공종별, 영역별로 나누어 설정하는 것이 관리하기 쉽습니다.

## 슬라이드 43. Save Selection과 Save Search

- Save Selection과 Save Search는 반복 검토를 효율적으로 하기 위한 기능입니다.
- Find Item에서 조건을 입력해 필요한 객체만 검색하고, 그 조건을 저장해두면 다음 검토 때 다시 사용할 수 있습니다.
- 검토 기준을 저장해두는 습관이 있어야 결과의 일관성을 유지할 수 있습니다.

## 슬라이드 44. Rule 설정

- Rule 설정은 조건부 간섭을 걸러내기 위한 단계입니다.
- 예를 들어 같은 파일 내부 객체나 특정 허용 조건을 제외하면 실제 문제에 더 집중할 수 있습니다.
- Rule을 너무 많이 적용하면 중요한 간섭을 놓칠 수 있으므로 기준을 명확히 하고 사용해야 합니다.

## 슬라이드 45. 간섭 항목 조건 변경

- 간섭 항목 조건을 변경하면서 실제 검토 결과가 어떻게 달라지는지 보여주는 슬라이드입니다.
- 초보자에게는 조건을 바꾸면 결과 수량이 달라진다는 점을 직접 확인시키는 것이 좋습니다.
- 단순히 간섭 개수만 보는 것이 아니라 실제 조치가 필요한 항목인지 판단해야 한다고 설명합니다.

## 슬라이드 46. 간섭검토 Report

- Report에 포함할 내용과 상태를 선택하는 단계입니다.
- 검토 보고서는 단순 캡처가 아니라, 어떤 간섭을 확인했고 어떤 상태인지 기록하는 산출물입니다.
- Contents와 Status를 적절히 선택해 검토자와 작업자가 같은 기준으로 결과를 볼 수 있게 해야 합니다.

## 슬라이드 47. 마무리

- 오늘 강의는 Revit 초급자가 BIM 모델링 업무의 전체 흐름을 이해하는 데 초점을 맞췄습니다.
- BIM 개념, Revit 화면, 공동작업, 패밀리, 링크, 트레이, 속성정보, 물량산출, 납품, 간섭검토가 서로 연결된다는 점을 다시 정리해 주세요.
- 마지막에는 실제 프로젝트에서는 한 번에 완벽히 하려고 하기보다, 기준을 정하고 반복 검토하는 습관이 중요하다고 마무리합니다.
