조도 계산을 엑셀 왕복 방식으로 운영하는 권장 흐름입니다.

기본 순서는 아래 4단계입니다.
1. `MEP Space` 값을 1차 export
2. 엑셀에서 `광원고`, `조도_광속_lm`, `조도_적용전등타입` 수동 입력
3. 수정한 엑셀을 Revit으로 import
4. 조도 계산 실행 후 결과를 다시 CSV로 2차 export

1. `MEP Space` 카테고리 기준으로 아래 공유파라미터를 준비합니다.
`필수 요구조도`, `조명율`, `보수율`, `광원고`, `조도_적용전등타입`, `조도_광속_lm`, `조도_실지수`, `필요등수`, `fixtureFlux_lm`, `조도_적용기구`

반사율은 입력하지 않고 고정값으로 사용합니다.
천장 `70`, 벽 `50`, 바닥 `10`

2. 엑셀 Export는 기존 [shared_parameter_values_export.py](C:/Users/user/Desktop/codex/#SharedParameterValues_Export/shared_parameter_values_export.py)를 사용합니다.
카테고리는 `MEP Space`만 선택하고, 파라미터 목록은 아래처럼 넣으면 됩니다.

```text
필수 요구조도, 조명율, 보수율, 광원고, 조도_적용전등타입, 조도_광속_lm, 조도_실지수, 필요등수, fixtureFlux_lm, 조도_적용기구
```

3. 엑셀에서 직접 입력할 컬럼은 아래 3개를 우선으로 봅니다.
`광원고`, `조도_적용전등타입`, `조도_광속_lm`

4. 엑셀 Import는 기존 [shared_parameter_values_import.py](C:/Users/user/Desktop/codex/#SharedParameterValues_Import/shared_parameter_values_import.py)를 사용합니다.
`ElementId` 또는 `UniqueId`가 유지된 상태로 다시 가져오면 `MEP Space` 파라미터에 값이 반영됩니다.

5. 계산 실행은 [@lighting_fixture_required_count.py](C:/Users/user/Desktop/codex/#조도계산서/@lighting_fixture_required_count.py)를 사용합니다.
이 스크립트는 아래 우선순위로 광속을 읽습니다.
`Space.조도_광속_lm` -> `Space.fixtureFlux_lm` -> 모델의 실제 Lighting Fixture 타입 광속

`광원고`는 Revit의 `MEP Space` 파라미터에서 직접 읽으므로, 1차 export 후 엑셀에서 수동 입력한 값을 import해야 실지수와 필요등수 계산이 가능합니다.

6. 계산 후 결과는 아래처럼 다시 `Space`에 기록됩니다.
`조도_실지수`, `필요등수`, `fixtureFlux_lm`, `조도_적용기구`

CSV 결과표에는 반사율이 아래 고정값으로 기록됩니다.
천장 `70`, 벽 `50`, 바닥 `10`

7. 최종 산출은 두 방식 중 하나로 가져가면 됩니다.
`Space Schedule`로 바로 표출
현재 계산 스크립트가 만드는 CSV로 외부 제출용 저장

권장 실행 순서:
1. `MEP Space` 파라미터 1차 export
2. 엑셀에서 `광원고`, `조도_광속_lm`, `조도_적용전등타입` 입력
3. 수정한 엑셀 import
4. 조도 계산 Dynamo 실행
5. 계산 결과 CSV 2차 export 확인

주의:
- 엑셀에서 `조도_광속_lm`을 입력하면 모델 내 전등 타입 매칭이 실패해도 계산은 진행됩니다.
- `조도_적용전등타입`은 기록용과 추적용 의미가 크고, 실제 광속 계산값은 `조도_광속_lm`이 우선합니다.
- `ElementId`, `UniqueId`, `Category`, `Family`, `Type` 헤더는 import 매칭용이므로 삭제하지 않는 것이 좋습니다.
